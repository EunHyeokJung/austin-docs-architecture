# 3) 서비스 기획안 (예: TaskFlow)

> ⚠️ 이 문서의 내용은 예시입니다. 실제 프로젝트에 맞게 수정하세요.

본 문서는 "TaskFlow" 서비스의 기획안(요구사항, 사용자 흐름, 기능 범위)을 정리합니다.  
현재는 **MVP 1차**를 목표로 합니다.

---

## A. 서비스 개요

### 목적
- 소규모 팀의 업무 관리와 진행 상황 추적을 한 곳에서 처리
- 직관적인 칸반 보드 기반 태스크 관리 제공

### 대상 사용자
- 스타트업/소규모 팀 리더
  - 팀원별 업무 현황을 한눈에 파악하고 싶음
- 개발자/디자이너 (팀원)
  - 자신의 태스크를 빠르게 확인하고 상태를 업데이트하고 싶음

---

## B. 서비스 구성 (App)

### 1) 일반 유저용 Web
- 대시보드 (칸반 보드 + 목록 뷰)
- 태스크 생성/수정/삭제
- 팀 멤버 초대 및 권한 관리

### 2) 관리자용 Web (추후)
- 전체 프로젝트 통계 대시보드
- 팀/멤버 관리

---

## C. 페이지 구성 (핵심 라우트 기준)

### 대시보드 (/dashboard)
- 칸반 보드: To Do / In Progress / Done 컬럼
- 태스크 카드 드래그 앤 드롭
- 필터: 담당자, 우선순위, 마감일

### 태스크 상세 (/tasks/:id)
- 태스크 제목, 설명, 첨부파일
- 댓글/활동 로그
- 상태 변경, 담당자 지정

---

## D. 핵심 기능 요구사항 (MVP 중심)

### 1) 태스크 CRUD
- 제목, 설명, 우선순위(Low/Medium/High), 마감일 입력 필수
- 드래그 앤 드롭으로 상태 변경

### 2) 인증 및 팀 관리
- 이메일/소셜 로그인 (Google, GitHub)
- 팀 생성 후 초대 링크로 멤버 추가
